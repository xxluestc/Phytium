# GD32L233C 主控器（Master Controller）设计文档

## 1. 概述

本文档描述基于 **GD32L233C + FreeRTOS** 的主控器软件设计。主控器通过 LoRa 无线模块（ATK-MWCC68D）与最多 **10 个终端节点**通信，接收终端上报的 **6 种数据类型**，将 **10 周期状态数据（400 点）** 和 **故障波形数据**分别存储到芯片内部 Flash 的不同区域，并根据节点故障状态向终端下发指令。

- **接收**: 终端上报的 6 种数据类型（状态头 / 波形头 / 电源 / 节点原始数据 / 波形原始数据 / 故障列表）
- **存储**: 10 周期状态数据（400 点 × 16B = 6400B）存于 Flash 状态区，波形数据（int16[]）存于 Flash 波形区
- **指令下发**: 混沌加密后通过 LoRa 发送，带 ACK 确认 + 最多 3 次重发
- **采集指令**: 采样率 6000Hz / 15000Hz、最大 500ms 波形采集指令

## 3. 任务说明

### 3.1 master_recv_task — 主控接收任务

**优先级**: 3 | **堆栈**: 512 字 | **文件**: `app/task/master_recv.c`

从 LoRa 模块接收终端发来的数据，**混沌解密**后根据 `DataType_t` 分发到对应处理器。内部维护一个**接收状态机**（`RecvState_t`）用于连续累积多包数据。

**接收数据类型及处理方式**:

| 类型 | 枚举值 | 载荷格式 | 处理方式 |
|------|--------|---------|---------|
| `DATA_TYPE_STATUS` | 0x01 | `NodeUploadData_t`(11B) 或 `FaultUploadHeader_t`(15B) | 区分周期上传/故障上传，更新节点快照，标记 is_online |
| `DATA_TYPE_WAVE` | 0x02 | `WaveChunkHeader_t`(12B) | 擦除该节点波形 Flash 区，状态机进入 `RECV_WAVE` |
| `DATA_TYPE_POWER` | 0x03 | 预留 | 仅日志记录 |
| `DATA_TYPE_NODE_RAW` | 0x04 | `NodeSample_t` × N (每包 128B / 8 样本) | 累存到 `g_dl_buf.node_buffer[]`，满 400 点后整体写入 Flash 状态区 |
| `DATA_TYPE_FLASH_WAVE` | 0x05 | `int16_t` × N (每包 128B / 64 样本) | 逐包追加写入 Flash 波形区，满后标记 `has_wave_data` |
| `DATA_TYPE_FAULT_LIST` | 0x06 | `uint8_t` 故障有效性数组 | 统计有效故障数，日志输出 |

**状态机流程**:
```
IDLE ──STATUS(0x01)──→ 解析头, 设置 dl 参数, dl->active=1
 IDLE ──WAVE(0x02)────→ 解析头, 擦除旧波形, dl->active=1, state=RECV_WAVE
 IDLE ──其他 ─────────→ 单包处理完成

NODE_RAW(0x04) ──→ 累积到 dl->node_buffer
    满 400 点 → master_flash_save_node_data() → 写 Flash → dl->active=0 → IDLE

RECV_WAVE ──FLASH_WAVE(0x05)──→ 直接写 Flash 波形区
    满 sample_count → master_recv_wave_data() → 写 Flash → dl->active=0 → IDLE
```

**执行流程**:
```
while(1) {
    1. atk_mwcc68d_receive_data(lora_buf, &recv_len, 5000ms)
       超时或无数据 → dl->active=0, continue
    2. 提取 sync_code(4B) + data_type(1B) → 混沌解密
    3. 标记节点在线, 更新时间戳
    4. switch(data_type) 分派到 process_xxx()
    5. vTaskDelay(50ms)
}
```

### 3.2 master_judge_task — 主控故障判断任务

**优先级**: 4 | **堆栈**: 256 字 | **文件**: `app/task/master_judge.c`

**周期**: 1000ms（每秒轮询所有 10 个节点）

**功能**:
- **离线检测**: 节点超过 15 秒无数据 → 标记 `is_online=0`
- **故障触发**: 节点在线且 `severity >= SEVERITY_WARNING`, `fault_type != FAULT_NONE`, `wave_pending == 0` → 向命令队列发送 `MASTER_CMD_REQ_WAVE`

**执行流程**:
```
while(1) {
    vTaskDelay(1000ms)

    for (i = 0; i < 10; i++) {
        if (now - node.last_recv_time > 15000ms)
            → 标记离线, log_warn, continue

        if (!在线 || wave_pending || fault_type==NONE) → continue
        if (severity < WARNING) → continue

        构建 MASTER_CMD_REQ_WAVE 指令
        cmd.node_id = i, cmd.fault_idx = fault_count-1
        cmd.sample_rate = 6000, cmd.duration_ms = 250
        → xQueueSend(g_master_cmd_queue, &cmd)
        → node->wave_pending = 1
    }
}
```

### 3.3 master_cmd_task — 主控指令下发任务

**优先级**: 5（最高） | **堆栈**: 256 字 | **文件**: `app/task/master_cmd.c`

从内部命令队列 `g_master_cmd_queue` 取出指令，**混沌加密 + LoRa 发送 → 等待 ACK**，最多重试 3 次。

**支持指令**:

| 内部指令 | 终端码 | 参数 | 说明 |
|---------|--------|------|------|
| `MASTER_CMD_REQ_WAVE` | `CMD_REQUEST_WAVEFORM` (0x10) | node_id(1B) + fault_idx(1B) | 请求终端上传指定节点的故障波形 |
| `MASTER_CMD_REQ_FAULT_LIST` | `CMD_REQUEST_FAULT_LIST` (0x11) | 无 | 请求终端上传故障记录列表 |
| `MASTER_CMD_CLEAR_FLASH` | `CMD_CLEAR_FLASH` (0x12) | 无 | 清除终端 Flash 波形存储区 |
| `MASTER_CMD_WAVE_COLLECT` | `CMD_START_WAVE_COLLECT` (0x13) | sample_rate(2B) + duration_ms(2B) | 指令终端启动新波形采集 |

**指令加密包格式**:
```
[sync_code(4B, 大端)][0x00(1B)][encrypted_payload(nB)]
```
- `sync_code`: 混沌加密前状态，终端据此恢复解密
- `encrypted_payload[0]`: 指令码, `[1..]`: 参数

**发送流程**:
```
send_lora_cmd():
    for retry in 0..2:
        chaos_encrypt_packet() → 加密载荷
        atk_mwcc68d_send_data() → LoRa 发送
        等待 ACK (2000ms)
        if 收到 ACK:
            node->wave_pending = 0, node->cmd_retry = 0
            return
        else:
            node->cmd_retry++, vTaskDelay(100ms)
    // 3次失败 → log_error, node->cmd_retry=0
```

## 4. 数据结构

### 4.1 数据类型枚举 (`DataType_t`) — `app/data_frame.h`

```c
typedef enum {
    DATA_TYPE_STATUS     = 0x01,   // 节点状态头 (NodeUploadData_t / FaultUploadHeader_t)
    DATA_TYPE_WAVE       = 0x02,   // 波形数据头 (WaveChunkHeader_t)
    DATA_TYPE_POWER      = 0x03,   // 电源电压数据（预留）
    DATA_TYPE_NODE_RAW   = 0x04,   // 节点原始数据包 (int32×4, 来自环缓冲)
    DATA_TYPE_FLASH_WAVE = 0x05,   // 波形原始数据包 (int16, 来自Flash存储)
    DATA_TYPE_FAULT_LIST = 0x06    // 故障记录列表
} DataType_t;
```

### 4.2 节点采样数据 (`NodeSample_t`) — `app/data_frame.h`

```c
typedef struct {
    int32_t active_power;      // 有功功率 ×10000
    int32_t reactive_power;    // 无功功率 ×10000
    int32_t voltage_angle;     // 电压相角 ×10000
    int32_t voltage_mag;       // 电压幅值 ×10000
} NodeSample_t;                // 共 16 字节
```

### 4.3 故障类型 (`FaultType_t`) — `app/data_frame.h`

```c
typedef enum {
    FAULT_NONE = 0,
    FAULT_OVER_VOLTAGE,      // 过压: Vrms > 1.05pu
    FAULT_UNDER_VOLTAGE,     // 欠压: Vrms < 0.95pu
    FAULT_VOLTAGE_SAG,       // 暂降: Vrms < 0.90pu
    FAULT_VOLTAGE_SWELL,     // 暂升: Vrms > 1.10pu
    FAULT_TRANSIENT          // 暂态突变（预留）
} FaultType_t;
```

### 4.4 故障级别 (`SeverityLevel_t`) — `app/data_frame.h`

```c
typedef enum {
    SEVERITY_NORMAL  = 0,
    SEVERITY_WARNING = 1,
    SEVERITY_DANGER  = 2
} SeverityLevel_t;
```

### 4.5 周期上传头 (`NodeUploadData_t`) — `app/data_frame.h`

### 4.8 节点信息快照 (`MasterNodeInfo_t`) — `app/task/master.h`

运行时常驻 RAM 的轻量级节点索引（每节点约 52B，10 节点共约 520B）：

```c
typedef struct {
    uint8_t      node_id;                // 节点号 0~9
    uint8_t      is_online;              // 在线标志
    uint8_t      has_status_data;        // Flash 中是否有有效 10 周期数据
    uint8_t      has_wave_data;          // Flash 中是否有有效波形数据
    uint8_t      severity;               // 当前故障级别
    FaultType_t  fault_type;             // 当前故障类型
    uint32_t     last_recv_time;         // 最后通信时间戳
    uint32_t     fault_count;            // 累计故障次数

    /* 最近一次 10 周期状态数据头 */
    uint8_t      last_status_type;       // 0=周期上传, 1=故障触发上传
    uint16_t     last_total_points;      // 总点数 (400)
    uint16_t     last_sample_rate;       // 采样率 (2000Hz)
    float        last_health_score;      // 健康度
    uint32_t     last_status_timestamp;  // 状态时间戳
    FaultType_t  last_status_fault;      // 状态中携带的故障类型

    /* 最近一次波形数据头 */
    uint8_t           has_last_wave_hdr;
    uint32_t          last_wave_rate;    // 波形采样率 (6000/12000Hz)
    uint32_t          last_wave_samples; // 波形采样点数
    SeverityLevel_t   last_wave_severity;

    /* 指令状态 */
    uint8_t      wave_pending;           // 是否有待处理的波形请求
    uint8_t      cmd_retry;              // 指令重试计数
} MasterNodeInfo_t;
```

### 4.9 RAM 下载缓冲区 (`MasterDownloadBuf_t`) — `app/task/master.h`

```c
    uint16_t     received_points;     // 已接收点数
    uint32_t     sample_rate;         // 采样率
    uint8_t      severity;            // 故障级别
    NodeSample_t node_buffer[FAULT_UPLOAD_POINTS];  // 400 × 16B = 6400B
} MasterDownloadBuf_t;
```

## 5. Flash 存储布局

状态数据和波形数据**物理隔离**，各自占据 64KB Flash 区域，擦除操作互不影响。

```
0x08000000 ┌────────────────────────────┐
           │  程序代码 (128KB)          │
0x08020000 ├────────────────────────────┤  ← 终端侧波形区 (128KB, 终端使用)
           │  (主控不使用此区域)         │
0x08030000 ├────────────────────────────┤  ← MASTER_STATUS_FLASH_BASE
           │  Node0 状态 [6400B]        │
           │  Node1 状态 [6400B]        │
           │  ...                       │
           │  Node9 状态 [6400B]        │
           │  共 64KB                   │
0x08040000 ├────────────────────────────┤  ← MASTER_WAVE_FLASH_BASE
           │  Node0 波形 [~6400B]       │
           │  Node1 波形 [~6400B]       │
           │  ...                       │
           │  Node9 波形 [~6400B]       │
           │  共 64KB                   │
0x08050000 └────────────────────────────┘
```

| 区域 | 基址 | 每节点 | 总容量 | 用途 |
|------|------|--------|--------|------|
| 状态数据区 | `0x08030000` | `0x1900` (6400B) | 64KB | 存储 400 点 `NodeSample_t`（10 周期 × 40 点/周期） |
| 波形数据区 | `0x08040000` | ~6400B（可变） | 64KB | 存储 `int16_t[]` 波形采样（6kHz×250ms=1500点 或 12kHz×400ms=4800点） |

**地址计算公式**:
```c
// 状态数据
addr = MASTER_STATUS_FLASH_BASE + node_id * MASTER_FLASH_PER_NODE

// 波形数据
addr = MASTER_WAVE_FLASH_BASE + node_id * MASTER_FLASH_PER_NODE
```

**Flash 读写接口** (`app/task/master_sys.c`):

| 函数 | 说明 |
|------|------|
| `master_flash_save_node_data(node, data, count)` | 保存 10 周期状态 → Flash |
| `master_flash_load_node_data(node, buf, max)` | 从 Flash 读取状态 → RAM |
| `master_flash_erase_node(node)` | 擦除节点状态区 |
| `master_flash_save_wave_data(node, data, len, offset)` | 追加保存波形 → Flash |
| `master_flash_load_wave_data(node, buf, len)` | 从 Flash 读取波形 → RAM |
| `master_flash_erase_wave(node)` | 擦除节点波形区 |

## 6. 通信流程

### 6.1 终端周期上传 10 周期状态数据

```
终端                                   主控器
  │                                      │
  │── STATUS(NodeUploadData_t, 11B) ──→│ master_recv: process_status_header()
  │  data_type=0x01, severity=NORMAL    │   → 记录 node_id, total_points=400
  │  node_index=3, total_points=400     │   → dl->expected_points = 400
  │←── ACK ────────────────────────────│   → dl->active = 1
  │                                      │
  │── NODE_RAW(0x04) × 50 包 ─────────→│ process_node_raw()
  │  每包 8 × NodeSample_t(128B)        │   → 累积到 dl->node_buffer[]
  │  共 400 点 (6400B)                   │
  │←── ACK (每包) ─────────────────────│
  │                                      │
  │                          收满 400 点 → master_flash_save_node_data()
  │                              → Flash 0x08030000 + node×0x1900 + 64B
  │                              → node->has_status_data = 1
```

### 6.2 故障触发 → 请求波形上传

```
终端                                   主控器
  │                                      │
  │── STATUS(FaultUploadHeader_t,15B)──→│ master_recv: process_status_header()
  │  severity=WARNING, type=暂降         │   → 更新 node->severity, node->fault_type
  │←── ACK ────────────────────────────│
  │                                      │
  │                              ┌───────┴──────────┐
  │                              │  master_judge:   │
  │                              │  检测 severity≥   │
  │                              │  WARNING, 发送    │
  │                              │  REQ_WAVE 到队列  │
  │                              └───────┬──────────┘
  │                                      │
  │                              ┌───────┴──────────┐
  │                              │  master_cmd:     │
  │←── CMD_REQUEST_WAVEFORM ────│  加密 → LoRa →   │
  │    (0x10) 加密包             │  等 ACK (3次重发) │
  │──→ ACK ──────────────────────│                   │
  │                              └───────────────────┘
  │                                      │
  │── WAVE(WaveChunkHeader_t, 12B) ──→│ master_recv: process_wave_header()
  │←── ACK ────────────────────────────│   → 擦除旧波形 Flash, state=RECV_WAVE
  │                                      │
  │── FLASH_WAVE(0x05) × N 包 ────────→│ process_flash_wave()
  │  每包 64 × int16_t(128B)            │   → 逐包追加写 Flash 波形区
  │←── ACK (每包) ─────────────────────│
  │                                      │
  │                          收满 sample_count → master_recv_wave_data()
  │                              → node->has_wave_data = 1
  │                              → node->wave_pending = 0
```

| `MASTER_FLASH_PER_NODE` | `0x1900` (6400B) | `master.h` | 每节点 Flash 空间 |
| `NODE_SAMPLE_RATE` | 2000 Hz | `data_frame.h` | 节点采样率 |
| `FAULT_UPLOAD_POINTS` | 400 | `data_frame.h` | 10 周期总点数 |
| `MAX_ENCRYPT_DATA_LEN` | 128 | `chaos_encrypt.h` | 加密数据最大长度 |

| 指令码 | 名称 | 方向 | 说明 |
|--------|------|------|------|

| `0x11` | `CMD_REQUEST_FAULT_LIST` | 主控→终端 | 请求上传故障记录列表 |

| `0x13` | `CMD_START_WAVE_COLLECT` | 主控→终端 | 启动指定采样率的波形采集 |
